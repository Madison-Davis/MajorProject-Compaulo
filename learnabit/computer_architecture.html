CHAPTER 02

Speculation

Execution Time = Instructions per Program × Cycles per Instruction (CPI) × Seconds per Cycle

Modern processors see relatively small improvements from compiler optimizations or clock-frequency scaling. Instead, most modern single-threaded performance gains come from reducing CPI through smarter microarchitecture techniques such as out-of-order execution, branch prediction, prefetching, and cache optimization.

Pipelines improve throughput by overlapping work across instructions, but real hardware is rarely single-cycle. Operations such as cache misses or floating-point division may require tens or even hundreds of cycles. To address this, processors implement multi-cycle execution units that split execution into multiple stages.

Multi-cycle execution introduces several new hazards. Structural hazards occur when multiple instructions compete for the same hardware resources. Data hazards such as RAW and WAW dependencies become more difficult because instructions may complete out of order. Long latencies also complicate exception handling and pipeline correctness.

Precise interrupts became a major architectural requirement in modern processors. When an exception occurs, the processor must guarantee that all older instructions appear completed while younger instructions appear untouched. Achieving this in out-of-order processors requires rollback, instruction squashing, and careful state management.

These challenges motivated the development of Instruction-Level Parallelism (ILP). ILP attempts to execute multiple independent instructions simultaneously in order to increase instructions-per-cycle (IPC) and reduce pipeline stalls.

Dynamic scheduling techniques allow hardware to reorder instructions at runtime. Unlike static compiler scheduling, hardware scheduling can react to real memory conditions, speculate on branches, and utilize additional internal registers. Common ILP techniques include scoreboarding, Tomasulo’s algorithm, register renaming, and superscalar execution.

One of the earliest ILP mechanisms was scoreboarding, introduced in the 1960s. Scoreboarding tracks instruction status, functional units, and register usage to safely support out-of-order execution while avoiding WAR and WAW hazards. Although effective for correctness, scoreboarding still suffers from performance limitations caused by RAW dependencies, structural bottlenecks, and scoreboard resource constraints.

Overall, modern processor design focuses less on increasing clock speed and more on extracting parallelism, hiding latency, and maximizing useful work per cycle. Much of contemporary CPU complexity exists because hardware aggressively attempts to execute instructions in parallel while still preserving correct program behavior.
